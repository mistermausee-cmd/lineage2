## 0. Краткая сводка (TL;DR)

- Клиент — **Lineage 2 Grand Crusade, протокол 110**, база **английская** («Tale of Aden
  Grand Crusade» с lineage2.org.uk), поверх неё наложена **русская папка `system`**.
- Версия клиента: `EP30_Global, V2110409, 110`, билд август 2017. Железо тестера: ноут RTX 4060.
- **Сервер (L2J Mobius GC) требует именно РУССКИЙ набор данных** (`ItemName-RU`, `Skillgrp`
  и т.д.). Английскую базу данных сервер НЕ принимает (ошибка «повреждён системный файл»).
  Поэтому рабочая система — **`system-ru`**, а не `system-e`.
- Клиентские файлы лежат в ветке **`system`** репозитория (папки `system-ru`, `system-e`,
  `SysTextures`). Серверные — в `main`.
- Все клиентские `.dat/.ini/.utx` **зашифрованы**; инструмент — **`l2encdec`** (open-l2encdec).

│                               NWindow.dll, Window.dll, ALAudio.dll, Fire.dll ...)
├── *.u                      ← Unreal-пакеты кода (Engine.u, NWindow.u, Interface.u)

├── *grp.dat                 ← данные предметов/нпц/навыков (Armorgrp, Weapongrp, EtcItemgrp,
│                               Skillgrp, Npcgrp ...) (Ver413)
├── ItemName-RU.dat / -e.dat ← названия+описания предметов (локализация) (Ver413)
├── *-RU.dat / *-e.dat       ← прочая локализация (NpcName, SkillName, SystemMsg, QuestName ...)
├── MinimapRegion.dat        ← карта: регион → тайлы (Ver413)
├── Localization.ini         ← ВЫБОР ЯЗЫКА + текстуры карты/логотипов по языкам (Ver111)
├── L2.ini, Option.ini, ...  ← конфиги (Ver111 / Ver413)
├── SmallFont-ru.gly, LargeFont-ru.gly, *-e.gly  ← глифы шрифтов
└── (текстуры .utx лежат в отдельной папке systextures клиента, НЕ в system/)

Текстуры (`.utx`) физически в папке `systextures` клиента (в репо — частично в `SysTextures/`).
Смена папки `system` НЕ меняет `systextures` — текстуры общие для любой системы.

## 2. Шифрование/расшифровка клиентских файлов

### Инструмент
**open-l2encdec** — https://github.com/ritsuwastaken/open-l2encdec (C++).
Сборка в рабочей среде: `/projects/sandbox/l2work/open-l2encdec/build/cli/l2encdec`
```bash
# сборка (если нужно заново):
git clone https://github.com/ritsuwastaken/open-l2encdec
cd open-l2encdec && cmake -B build && cmake --build build
```

### Как определить версию файла
У зашифрованного файла первые байты — UTF-16LE строка **`Lineage2Ver<NNN>`**.
```bash
python3 -c "print(open('L2GameDataName.dat','rb').read()[:28].decode('utf-16-le','replace'))"
# → 'Lineage2Ver413'
```

### Версии (протоколы шифрования) и команды
| Ver | Алгоритм | Где встречается |
|-----|----------|-----------------|
| **111** | XOR | `Localization.ini`, `Option.ini` |
| **120** | XOR по позиции | (редко) |
| **121** | XOR по имени файла | текстуры `.utx` (напр. `L2_RadarMap.utx`) |
| **411–414** (обычно **413**) | RSA (+ deflate внутри) | `.dat`: L2GameDataName, ItemName, *grp, MinimapRegion |

```bash
L2E=/projects/sandbox/l2work/open-l2encdec/build/cli/l2encdec
# ДЕКОД (версия определяется автоматически, но лучше указать -p):
$L2E -c decode -p 413 -o out.dec  file.dat
# ЭНКОД (протокол указывать ОБЯЗАТЕЛЬНО):
$L2E -c encode -p 413 -o file.dat out.dec
# другие: -p 111 для .ini, -p 121 для .utx
```

### ВАЖНО про round-trip (потери)
- `decode(encode(x))` даёт **байт-в-байт то же расшифрованное содержимое** → клиент принимает.
- НО перекодированный зашифрованный файл может отличаться по размеру от оригинала
  (другая компрессия/хвост). Это НОРМАЛЬНО — клиент читает то же содержимое.
- Всегда проверяй round-trip перед установкой: `decode(файл-который-ставим)` == задуманное.
- `Ver413` включает deflate: расшифрованный L2GameDataName ~в 8 раз больше зашифрованного.
- Флаг `-t` — без хвоста (для файлов вроде Exteel). Обычно не нужен.

## 3. Форматы ключевых файлов

### 3.1 `L2GameDataName.dat` (Ver413) — таблица строк-имён
Глобальный справочник **внутренних имён ассетов** (пакеты текстур, меши), НЕ отображаемых
названий предметов. На него ссылаются ПО ИНДЕКСУ (номер записи) файлы `*grp.dat`,
`MinimapRegion.dat` и др.
```
[u32 count]
count × [u32 len][UTF-16LE строка длиной len]
[хвост-маркер "SafePackage" как len-префиксная строка]  ← НЕ входит в count!
```
- Кол-во: ~90 790 (РУ) / ~95 331 (EN).
- **Изменение содержимого записи НЕ сдвигает индексы** (индекс = порядковый номер),
  поэтому переименовывать строки на месте безопасно; **добавлять** можно только в конец.
- Парсер/пример: см. `fix_map_gdn.py`.

### 3.2 `ItemName-RU.dat` / `ItemName-e.dat` (Ver413) — названия предметов
```
[u32 count ~41000] затем записи: id + поля + строки (name, desc, action)
```
- РУ — текст в **UTF-16LE**; EN — в однобайтовой кодировке (latin1) → RU-файл почти вдвое больше.
- Перекрёстно связан с навыками. При загрузке сервер шлёт «alternate data» и сверяет
  согласованность ItemName ↔ SkillName ↔ Skillgrp (см. §6).

### 3.3 `*grp.dat` (Ver413) — данные предметов (Armorgrp, Weapongrp, EtcItemgrp, Skillgrp…)
- Ссылки на меши/текстуры — по индексам в `L2GameDataName.dat`.
- **Armorgrp.dat**: ~10 891 (EN) / ~10 901 (РУ) записей, файл заканчивается маркером
  **SafePackage** (в коде `SAFE`). Байт-точный парсер: `/projects/sandbox/l2work/armorgrp.py`
  (структура взята из инструмента **l2w_tool**, Cryme/l2w_tool, проверена round-trip).

### 3.4 `MinimapRegion.dat` (Ver413) — большая карта
```
[u32 count=203]  # регионов
на регион: regionId(u16) + индексы текстур(u32 в L2GameDataName) + координаты + индексы шрифта-подписи
```
- Ссылается на: `L2font-<lang>.WorldIcon.*` (тайлы/маркеры карты), `L2UI_CT1.Minimap.*`
  (иконки фракций), `MapFontSize8/9/10` (шрифты подписей регионов).
- Тайлы именуются по координатной сетке: `19_22_0`, `20_14_0` и т.д. (плюс `_a` — альфа-слой,
  `_castle`/`_camp`/`Fort_common` — маркеры).

### 3.5 `Localization.ini` (Ver111) — ЯЗЫК и текстуры карты/лого
Ключевой файл! Определяет язык и КАКИЕ текстуры грузить для карты/логотипов.
```ini
[LanguageSet]
Language=8            ; 1=English (код 'e'), 8=Russia (код 'ru')
...
[Russia]
Russia_Code=ru
Russia_Font=L2Font-ru.SmallFont-ru
Russia_Glyph=SmallFont-ru.gly
Russia_Minimap=L2Font-e.Minimap.worldmap   ; ← ФОН большой карты (worldmap)
Russia_TownMap=L2Font-e.TownMap            ; ← карты городов
Russia_LogoTexture=L2Font-e.start_logo-e   ; логотип/лого-загрузки
...
```
- `Language=N` выбирает язык → клиент грузит `<Файл>-<код>.dat` (напр. `ItemName-RU.dat`).
- В каждой секции языка: шрифт, глиф, **Minimap (фон карты)**, TownMap, логотипы.

### 3.6 `.utx` (Ver121) — текстурные пакеты (Unreal)
- После расшифровки — UE2-пакет: сигнатура, версия пакета, таблицы Names/Exports/Imports.
- Чтение имён: см. пример в истории (парс таблицы Names по compact-index).
- `L2_RadarMap.utx` = тайлы **радара** (мини-карта в углу): `RadarMap_1..20` + карты городов.
- `L2Font-e.utx` (~205 МБ) = группы `WorldIcon` (тайлы большой карты), `Card`, `Loading`,
  `Minimap.worldmap` (фон), шрифты. **`L2Font-ru.utx` в этом клиенте ОТСУТСТВУЕТ.**
- Полноценно распаковывать/пересобирать `.utx` — только сторонним **UModel/l2tool**, не l2encdec.

### 3.7 `Interface.xdat` — окна UI (НЕ зашифрован)
- Формат: `[u32]` + записи окон `[u8 len][ASCII имя][бинарные данные + встроенная XML-разметка]`.
- У окна есть позиция (X,Y) и размер рамки (W,H) как int32. **Размер содержимого
  (иконки 32×32, шрифты) хранится отдельно и ФИКСИРОВАН** — масштабировать контент через
  xdat НЕЛЬЗЯ (см. §8).
- Пара к нему — `Interface.u` (скомпилированный код UI, Ver-шифрован).

## 4. Как работает выбор языка

1. `Localization.ini` → `Language=8` (Россия).
2. Клиент грузит локализацию `*-RU.dat` (код `ru`; на Windows регистр не важен, `-RU`≈`-ru`).
3. Шрифты: `SmallFont-ru.gly`, `LargeFont-ru.gly` (глифы). Текстура шрифта `L2Font-ru.*`
   отсутствует, но текст рендерится (глифы + фолбэк) — русский работает.
4. Текстуры карты/лого берутся из секции `[Russia]` — вот тут была проблема (см. §5).

Русский текст = клиентские `-RU.dat`. Серверные HTML (Alt+B, NPC-диалоги) переводятся отдельно
на стороне сервера.

## 5. Карта: как устроена и как чинили «чёрную карту»

Большая карта (открывается кнопкой) состоит из ДВУХ слоёв из РАЗНЫХ источников:
| Слой | Источник | Управляется |
|---|---|---|
| **Фон местности** (`worldmap`) | `L2Font-<lang>.Minimap.worldmap` | `Localization.ini` → `<lang>_Minimap` |
| **Маркеры/тайлы-оверлеи** (крепости, города, зоны) | `L2font-<lang>.WorldIcon.*` | `MinimapRegion.dat` (индексы в `L2GameDataName`) |

Радар в углу — отдельно, из `L2_RadarMap.utx` (работал всегда).

### Причина «чёрной карты» на РУ-системе
В клиенте **нет `L2Font-ru.utx`**, а РУ-система ссылалась на него и для фона, и для тайлов:
- фон: `Russia_Minimap=L2Font-ru.Minimap.worldmap` → пакета нет → чёрный фон;
- тайлы: `MinimapRegion` → `L2font-ru.WorldIcon.*` → пакета нет → нет маркеров.

Английская система работала, т.к. `Language=1` → `English_Minimap=L2Font-e.Minimap.worldmap`
и `L2font-e.WorldIcon.*`, а `L2Font-e.utx` в клиенте ЕСТЬ.
**Это НЕ проблема видеокарты/рендера — это отсутствующий пакет текстур.**

### Как починили (две правки, обе в `system-ru`)
1. **Маркеры** — в `L2GameDataName.dat` переименованы 171 записи
   `L2font-ru.WorldIcon.*` → `L2font-e.WorldIcon.*` (на месте, индексы не сдвинуты).
   Скрипт: `/projects/sandbox/l2work/fix_map_gdn.py`.
2. **Фон** — в `Localization.ini` секция `[Russia]`: `Russia_Minimap`, `Russia_TownMap`,
   логотипы переключены с `L2Font-ru.*` на существующий `L2Font-e.*`. Шрифты (`SmallFont-ru.gly`)
   НЕ трогали — русский текст сохранён. Скрипт: `/projects/sandbox/l2work/fix_loc_ru.py`.

> На заметку: `L2font` (строчная f) в этом наборе содержал ТОЛЬКО группу `WorldIcon`.
> Русские шрифты/карточки — в другом пакете `L2Font-ru` (заглавная F), его не трогали.

## 6. Совместимость с сервером (что ломает вход!)

При входе сервер шлёт `VersionCheckPacket` → `LoadAlternateGameData` → `MSItemNameDataLoad`
и **сверяет согласованность** клиентских данных предметов/навыков. Если клиентские
`ItemName`/`SkillName`/`Skillgrp` не соответствуют серверному набору — ошибка:
> «Есть вероятность, что был повреждён системный файл…» (в логе: `OverwriteSkillData skill ID [...]`).

**Вывод:** данные (`*grp.dat`, `ItemName-*`, `SkillName-*`, `L2GameDataName`, `MinimapRegion`)
должны быть из **русского набора** — под него собран сервер. Английский набор данных вход НЕ проходит
(проверено: EN `Skillgrp.dat` 819376 б ≠ РУ 819504 б → рассинхрон навыка 28142 → ошибка).

| Файл | Проверяется сервером при входе? | Можно ли брать из EN? |
|---|---|---|
| `Interface.u`, `Interface.xdat` | ❌ нет (чисто UI) | ✅ да, безопасно |
| `Localization.ini` | ❌ нет | ✅ да (мы и брали из него `L2Font-e` для карты) |
| `L2GameDataName.dat`, `*grp.dat`, `ItemName-*`, `SkillName-*`, `MinimapRegion` | ✅ ДА | ❌ нет — только РУ |
| `.dll`, `l2.exe` | частично (защита/протокол) | ⚠️ РУ-бинарники надёжнее для входа |

**Итог по стратегии:** базой берём **РУ-систему** (совместима с сервером) и точечно чиним
клиентские мелочи. Английскую базу под этот сервер использовать НЕЛЬЗЯ.

## 8. Масштаб интерфейса (почему UI мелкий на высоком разрешении)

- В клиенте GC **нет встроенного масштаба UI**. Иконки — фикс. картинки 32×32, шрифты фикс.
- Правка размеров окон в `Interface.xdat` **НЕ масштабирует содержимое** — только рамку
  (получится большое окно с мелкими иконками, сетка инвентаря может сломаться).
- Реальные способы «крупнее»:
  1. Играть в меньшем разрешении (напр. 1536×960) в полный экран — монитор растянет.
  2. **DPI-масштаб Windows** (напр. 125% = ровно коэффициент 1536→1920) + у `l2.exe`:
     Свойства → Совместимость → Изменить параметры высокого DPI → «Масштабирование: Система».
     Играть в оконном/безрамочном. UI станет крупнее (чуть мягче).
  3. Крупно И чётко на нативном разрешении — только **кастомный HD-интерфейс** (с перерисованными
     2× иконками + увеличенными шрифтами). Правкой xdat не делается. Файлы UI можно менять
     безопасно (сервер их не проверяет, §6) — источник брать у сообщества сервера / maxcheaters,
     обязательно под **протокол 110**.

## 9. Инструменты и рабочие файлы (среда `/projects/sandbox/l2work/`)

| Файл | Назначение |
|---|---|
| `open-l2encdec/build/cli/l2encdec` | шифрование/расшифровка (см. §2) |
| `armorgrp.py` | байт-точный парсер Armorgrp.dat |
| `hybrid.py` / `hybrid_e.py` | сборка гибридного Armorgrp (плащи) для РУ / EN базы |
| `fix_map_gdn.py` | переименование WorldIcon в L2GameDataName (маркеры карты) |
| `fix_loc_ru.py` / `fix_loc_e_merge.py` | правка Localization.ini (фон карты / язык) |
| `l2w_tool/` | Cryme/l2w_tool — референс структур .dat |
| `*.backup.*` | бэкапы оригиналов перед правками |

Стандартный цикл правки любого `.dat`:
```
1. decode → 2. распарсить структуру → 3. изменить → 4. re-serialize (проверить байт-точность
   на неизменённых записях) → 5. encode → 6. decode обратно и сверить → 7. поставить в system-ru
   → 8. commit + push в ветку system.
```

## 10. Доступ к репозиторию из рабочей среды

- Прямой `git fetch/push` по HTTPS падает («Missing header field AuthToken»).
- Используем **github power**: `pull_repository` (тянет), `push_to_remote` (пушит,
  `path=/projects/sandbox/lineage2`), `clone_repository`.
- Клиентские правки → ветка **`system`** (папки `system-ru`/`system-e`).
- Файлы больше 100 МБ тестер заливает zip-архивами по ~70 МБ (в т.ч. `.utx` текстуры).
- Во фронтенде видны только `.md` из `.kiro/specs/`; всё остальное смотрим/отдаём через GitHub.
