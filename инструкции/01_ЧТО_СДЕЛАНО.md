# Инструкция 1. Что уже сделано (полная история изменений)

> Документ-передача для будущих чатов. Здесь — ВСЁ, что было изменено, с путями и значениями.
> Репозиторий: **github.com/mistermausee-cmd/lineage2**

## 0. Основа
- Движок: **L2J Mobius**, хроника **Grand Crusade** (папка исходников `L2J_Mobius_04.0_GrandCrusade` с GitLab `MobiusDevelopment/L2J_Mobius`).
- Собирается: **Apache Ant** + **JDK 25** (`source/target=25`). Команда: в папке проекта `ant` → создаёт zip-дистрибутив, jar-ы (`LoginServer.jar`, `GameServer.jar`) в `server/libs/`.
- Сервер = 2 процесса: **LoginServer (порт 2106)** + **GameServer (порт 7777)**, общая БД **MySQL `l2jmobius`**.

## 1. Ветки репозитория
- **`main`** — сам сервер + конфиги + скрипты + документация + генератор магазина.
- **`system`** — клиентские файлы (НЕ трогать при работе с сервером):
  - `system.zip` + `system.z01` — чистая английская системка (протокол 110).
  - `system ruoff/` — официальная РУССКАЯ системка (ruoff), `.dat` с русским текстом.
  - `system new/` (`system-ru.zip`+`.z01`) — готовый РУ-клиент v110 (с `l2.exe`, `NOGG=true`), пак от l2-scripts.ru.
  - `system-ru-overlay.zip` — русские `.dat` (продублированы как `-e` и `-RU`) для наложения на клиент.
  - `l2scripts/` — исходники L2Scripts GoD 32081 (для сравнения магазина, НЕ используется как движок).
  - `geodata/` — геодата Grand Crusade от L2Script (220 файлов `.l2j`, ~820 МБ). Ставится на сервере через `server/game/download_geodata.sh` (в ветке `main` геодата не хранится).
  - `РУСИФИКАЦИЯ.md`, `logs` — вспомогательное.

## 2. Баланс (все правки — в `server/game/config/`)
**Rates.ini:**

**Custom/NpcStatMultipliers.ini** (ослабление РБ под соло):
- `EnableNpcStatMultipliers = True`
- `RaidbossHP = 0.35`, `RaidbossPAtk/MAtk = 0.55`, `RaidbossPDef/MDef = 0.6`
- Обычные мобы/стражи/защитники = `1.0` (мир не ослаблен)
- ⚠️ Множитель НЕ действует на GRAND-боссов (эпики) — их статы отдельно, ещё НЕ снижены.

**Player.ini:** `MaxSubclassLevel = 110` (мейн и так до 110, капа «99» в GC нет).

**Соло-инстансы:** во ВСЕХ файлах `server/game/data/instances/*.xml` условие `GroupMin` → `limit=1` (97 инстансов). Вход соло разрешён.

## 3. Сеть / запуск / стабильность
- **Server.ini:** `AllowedProtocolRevisions = 109;110` (совместимость с RU-клиентом GC), `GameserverHostname = 0.0.0.0`.
- **GeoEngine.ini:** `PathFinding = 2` (ВКЛЮЧЕНО). Геодата подключена — Grand Crusade от L2Script, 220 регионов, хранится в ветке `system` (папка `geodata/`), качается скриптом `server/game/download_geodata.sh`. Подробности — в `07_ALT_B_МЕНЮ_РЕЙТЫ_ЭКСПЛУАТАЦИЯ.md` §8.
- **Память:** `server/game/java.cfg` → `-Xmx8g -Xms2g` (с запасом, в т.ч. под геодату).
- **Папки логов:** `server/{game,login}/log/.gitkeep` + `start_linux.sh` создаёт их (`mkdir -p`) — иначе сервер не стартовал.

## 5.1 Остальное меню Alt+B (телепорт, склад, премиум, информация)

## 8. Как запускать/останавливать/обновлять сервер на VPS
```bash
# ОСТАНОВКА (правильная — гасит и циклы *Task.sh, и java):
cd ~/lineage2 && bash stop_linux.sh
# ⚠️ НЕ использовать только 'pkill -f *.jar' — циклы LoginServerTask/GameServerTask
#    тут же перезапустят сервер (это вызывало бесконечный краш-цикл и 400к лог-файлов).

# ОБНОВЛЕНИЕ (не затирает локальные Database.ini / ipconfig.xml):
git config core.fileMode false           # один раз: чтобы бит +x не мешал pull
git pull --ff-only origin main

# ЗАПУСК:
bash start_linux.sh
```
Подробности эксплуатации, логов и рейтов — в `07_ALT_B_МЕНЮ_РЕЙТЫ_ЭКСПЛУАТАЦИЯ.md`.

## 8.1 Правки этой сессии (фиксы багов + Теодор + предметы + ядро)

**A. Пересборка ядра (GameServer.jar)** — подробности и код в `08_ПЕРЕСБОРКА_ЯДРА.md`:
- Установлена ТОЧНАЯ версия исходника Mobius: ревизия **`88b6c5f4`** (master 2026-07-09), JDK 25, Ant 1.10.15.
- **Фикс бага колец богов** (серверный): на логине активные/ON_EQUIP скиллы эпик-бижи (тоглы колец)
  становились неактивными до переодевания. Добавлен `Item.reapplyActiveSkillsToOwner()`, вызывается из
  `PlayerInventory.applyItemSkills()` только на логине. Проверена вся бижа богов — корень общий, фикс общий.
- Добавлен `Item.setRemainingTime(long)` — выдача временных копий постоянных предметов (3-дня руны/бустеры
  Теодора) на ОРИГИНАЛЬНЫХ id, без кастомных id/клиентских записей.
- ⚠️ Пересборка меняет ТОЛЬКО jar. Все прежние правки конфигов/данных/скриптов/магазина сохраняются.
  Предметы игрока (в БД) не пропадают; предметы магазина — постоянные.
