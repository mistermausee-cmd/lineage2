# Инструкция 2. Архитектура сервера (с чем работаем)

> Для будущего чата: как всё устроено, где что лежит, как собирать и менять.

## Общая схема
Lineage 2 = **сервер (эмулятор)** + **клиент (игра на ПК игрока)**. Мы работаем с сервером **L2J Mobius** (Java, open-source) хроники **Grand Crusade**.

```
Клиент (l2.exe, протокол 110, РУ) ──► LoginServer (порт 2106) ──► выдаёт адрес game
                                   └► GameServer (порт 7777)  ◄── общая БД MySQL (l2jmobius)
```
- **LoginServer** — авторизация, список серверов. Порт **2106**.
- **GameServer** — сам игровой мир. Порт **7777**. Оба читают одну БД `l2jmobius`.
- **ipconfig.xml** (`server/game/config/`) — какой IP GameServer СООБЩАЕТ клиенту (это НЕ bind-адрес; bind = `0.0.0.0`). Для VPS = публичный IP.

## Что и как менять (типы изменений)
| Что меняем | Где | Нужна перекомпиляция? |
|---|---|---|
| Рейты, дроп, статы РБ, лимиты | `server/game/config/*.ini` | ❌ нет (рестарт) |
| Ассортимент/цены магазина | `data/multisell/6000XX.xml` (+ `tools/shopgen.py`) | ❌ нет |
| Меню Alt+B (кнопки/разделы) | `data/html/CommunityBoard/Custom/*.html` | ❌ нет |
| Условия входа в инстансы | `data/instances/*.xml` | ❌ нет |
| Спавны/дроп конкретных NPC | `data/stats/npcs/*.xml` | ❌ нет |
| Новая механика/сервис (профессии в Alt+B, топы, соло-оли, смена имени) | Java `.java` в `java/` | ✅ ДА (пересборка jar) |

## Как собрать сервер (перекомпиляция)
Нужны исходники Mobius (папка `L2J_Mobius_04.0_GrandCrusade` с `java/`, `dist/`, `build.xml`). В нашей рабочей среде они клонировались в `/projects/sandbox/mobius-src/`.
```bash
# 1. Исходники (если их нет):
git clone --filter=blob:none --sparse https://gitlab.com/MobiusDevelopment/L2J_Mobius.git mobius-src
cd mobius-src && git sparse-checkout set L2J_Mobius_04.0_GrandCrusade
# 2. Ant (если нет): скачать apache-ant, распаковать.
# 3. Сборка (JDK 25):
cd L2J_Mobius_04.0_GrandCrusade
JAVA_HOME=<путь к jdk25> <ant>/bin/ant       # → build/L2J_Mobius_04.0_GrandCrusade.zip
# 4. Распаковать zip → получить server/ (game, login, libs с новыми jar).
```
Правки конфигов/данных делаются в `dist/game/...` (в исходниках) ИЛИ прямо в `server/...` (в репо). После правки Java — обязательно пересборка.

## Магазин: как устроен
- `tools/shopgen.py` парсит `stats/items/*.xml`, по правилам (по грейдам/типам/именам) выбирает предметы, пишет `multisell/6000XX.xml` (валюта — адена, id 57) и генерирует HTML-меню отделов.
- Меню Alt+B: `merchant/main.html` (отделы) → `merchant/<dept>.html` (кнопки категорий) → `_bbsmultisell;<id>,merchant/<dept>` открывает мультиселл.
- Каждый мультиселл ОБЯЗАН иметь `<npcs><npc>-1</npc></npcs>` (разрешение открытия без NPC).

## Клиент
- РУ Grand Crusade, **протокол 110** (сервер принимает 109;110).
- Адрес сервера — в `system/l2.ini`, секция `[Auth]` → `ServerAddr`.
- `NOGG=true` — GameGuard отключён (нужно для приватки).
- Русский язык — из клиентских `.dat` (`ItemName-RU.dat` и т.д.). Серверные HTML переводятся отдельно.

## Среда (VPS)
- Azure Ubuntu 24.04, Java 25 (Temurin), MySQL 8.
- Порты 2106/7777 открыты в `ufw` И в Azure NSG.
- Navicat к БД — через SSH-туннель (MySQL слушает localhost).
